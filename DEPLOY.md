# 🚀 نشر FLX Workspace على الإنترنت — دليل مبسّط

هذا الدليل يأخذك من «المشروع على جهازي» إلى **رابط حقيقي على الإنترنت** خلال ~١٠ دقائق.
لا يحتاج خبرة برمجية. كل ما تكتبه هو نسخ ولصق.

> **مهم:** التطبيق محلياً يستخدم قاعدة بيانات SQLite، وهي **لا تعمل على الإنترنت**.
> لذلك في الإنترنت نستخدم قاعدة **PostgreSQL** مجانية (Neon). المشروع مُجهَّز لذلك تلقائياً.

---

## ما تحتاجه (كلها مجانية)
1. حساب على **Vercel** → https://vercel.com/signup (سجّل بحساب GitHub أو بريدك).
2. قاعدة بيانات مجانية على **Neon** → https://neon.tech (سجّل مجاناً).

---

## الخطوة ١ — احصل على قاعدة بيانات مجانية (Neon)
1. ادخل https://neon.tech واضغط **Create project**.
2. بعد الإنشاء، انسخ **Connection string** (يبدأ بـ `postgresql://...`).
   - اختر النسخة المكتوب بجانبها **Pooled connection** إن وُجدت.
3. احتفظ به — سنلصقه في Vercel بعد قليل. سمّه `DATABASE_URL`.

---

## الخطوة ٢ — انشر على Vercel

### الطريقة الأسهل (بدون GitHub) — عبر سطر الأوامر
افتح **Terminal** داخل مجلد `flx-app` ونفّذ:

```bash
npx vercel login        # سجّل الدخول (يفتح المتصفح)
npx vercel              # ارفع المشروع (أجب: Set up and deploy? → Yes، والباقي اضغط Enter)
```

ثم اذهب إلى لوحة Vercel → مشروعك → **Settings → Environment Variables** وأضف المتغيّرات (انظر الجدول أدناه)، ثم:

```bash
npx vercel --prod       # النشر النهائي بعد إضافة المتغيّرات
```

### الطريقة البديلة — عبر GitHub (زر واحد)
1. ارفع المجلد `flx-app` إلى مستودع على GitHub (عبر GitHub Desktop أبسط طريقة).
2. في Vercel اضغط **Add New → Project → Import** واختر المستودع.
3. أضف المتغيّرات (الجدول أدناه) واضغط **Deploy**.

> بعد رفعه على GitHub يمكنك أيضاً استخدام زر النشر السريع (بدّل الرابط برابط مستودعك):
> `https://vercel.com/new/clone?repository-url=<رابط_مستودعك_على_GitHub>`

---

## الخطوة ٣ — المتغيّرات البيئية (Environment Variables)

في Vercel → **Settings → Environment Variables** أضف التالي:

| المتغيّر | إلزامي؟ | القيمة |
|---------|--------|--------|
| `DATABASE_URL` | ✅ **نعم** | رابط Neon الذي نسخته في الخطوة ١ |
| `SESSION_SECRET` | ✅ **نعم** | أي نص عشوائي طويل (٣٠+ حرفاً). مثال: اكتب حروفاً وأرقاماً عشوائية |
| `APP_URL` | مُستحسن | رابط موقعك بعد النشر، مثل `https://flx-workspace.vercel.app` |
| `MAIL_FROM` | اختياري | `FLX Workspace <no-reply@flx.ps>` |
| `SMTP_HOST` `SMTP_PORT` `SMTP_USER` `SMTP_PASS` | اختياري | لإرسال بريد حقيقي (انظر «البريد» أدناه) |
| `WHATSAPP_TOKEN` `WHATSAPP_PHONE_NUMBER_ID` | اختياري | لتفعيل واتساب (انظر «واتساب» أدناه) |

> بعد إضافة المتغيّرات، أعد النشر (Redeploy) لتأخذ مفعولها.
> النشر يُنشئ الجداول ويزرع بيانات تجريبية **تلقائياً** (مرة واحدة فقط — لا يمسح بياناتك لاحقاً).

---

## الخطوة ٤ — افتح موقعك وجرّب
افتح الرابط الذي يعطيك إياه Vercel، وسجّل الدخول بأحد الحسابات:

| الحساب | المستخدم | كلمة المرور |
|------|---------|------------|
| عميل · يافا (فيديوهات حقيقية) | `yafa` | `yafa123` |
| عميل · جامعة النجاح | `lina` | `123456` |
| عميل · بنك فلسطين | `sami` | `123456` |
| مدير حساب (يرى الجميع) | `sara` | `123456` |
| مشرف عام | `admin` | `admin123` |

> 🔒 غيّر كلمات المرور هذه قبل أي استخدام حقيقي.

## البريد الإلكتروني 📧
- **بدون أي إعداد**: يعمل فوراً عبر صندوق اختبار (يعطيك رابط معاينة للرسالة).
- **لبريد حقيقي يصل لصندوق العميل**: أضف بيانات SMTP. أسهل خيار **Gmail**:
  1. فعّل التحقق بخطوتين في حساب Google.
  2. أنشئ **App Password** من إعدادات أمان Google.
  3. في Vercel ضع:
     - `SMTP_HOST=smtp.gmail.com`
     - `SMTP_PORT=465`
     - `SMTP_USER=بريدك@gmail.com`
     - `SMTP_PASS=رمز التطبيق (App Password)`

## واتساب 📱
- مُبرمَج بالكامل، يُفعَّل تلقائياً عند إضافة بيانات **WhatsApp Cloud API** من Meta:
  1. https://developers.facebook.com → أنشئ تطبيقاً → أضف **WhatsApp**.
  2. من **API Setup** انسخ **Temporary access token** و **Phone number ID**.
  3. في Vercel ضع `WHATSAPP_TOKEN` و `WHATSAPP_PHONE_NUMBER_ID`.
  4. أعد النشر → ستصل رسائل واتساب فعلية.
  > الرمز المؤقت من Meta ينتهي خلال ٢٤ ساعة؛ للإنتاج أنشئ **Permanent token**.

## ملاحظات مهمة
- **الفيديوهات**: فيديو «الوحش» حجمه ٦٠ ميغا. قد يبطّئ النشر أو يصطدم بحدود الخطة المجانية.
  إن واجهت مشكلة، احذفه من `public/clients/yafa/` أو استضِف الفيديوهات على خدمة فيديو
  (مثل Mux أو Cloudflare Stream) للأداء الأفضل.
- **البيانات التجريبية** تُزرع مرة واحدة عند أول نشر فقط، فلا تُمسح عند إعادة النشر.
- ملف `.env` لا يُرفع إلى GitHub (محمي) — ضع الأسرار في إعدادات Vercel فقط.
